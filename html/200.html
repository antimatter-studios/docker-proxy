Quick Help

How do I register a service?

Add docker labels to your container so Docker Config Gen can discover it:

services:
    website:
        build: .
        networks:
            - public
        labels:
            - docker-proxy.main.host=myapp.localhost
            - docker-proxy.main.port=3000

The only required label is host. Port defaults to 80, path to /, protocol to http.

Multiple upstreams per container

Use different upstream names in the label key: docker-proxy.{name}.host

labels:
    - docker-proxy.admin.host=myapp.localhost
    - docker-proxy.admin.port=8080
    - docker-proxy.admin.path=^/admin
    - docker-proxy.api.host=myapp.localhost
    - docker-proxy.api.port=5000
    - docker-proxy.api.path=^/api

Network requirements

The proxy container and your service must be on the same Docker network. A common pattern is to create a public network and add both containers to it.
